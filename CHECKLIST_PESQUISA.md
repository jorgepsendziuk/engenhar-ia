# Checklist de Pesquisa e Validação de Fontes

Este checklist garante que o conteúdo criado seja baseado em fontes confiáveis, atualizadas e validadas.

## Fase 1: Identificação do Tópico

- [ ] Tópico identificado no `course-structure.json`
- [ ] Subtópicos listados e compreendidos
- [ ] Objetivos de aprendizado definidos
- [ ] Pré-requisitos identificados

## Fase 2: Pesquisa de Fontes

### 2.1 Fontes Primárias (Obrigatórias)

- [ ] **Documentação Oficial**: Encontrada e revisada
  - [ ] Site oficial da tecnologia/ferramenta
  - [ ] Versão mais recente da documentação
  - [ ] Exemplos oficiais revisados

- [ ] **Papers Acadêmicos** (quando aplicável)
  - [ ] Paper original identificado
  - [ ] Paper lido e compreendido
  - [ ] Citações corretas incluídas

### 2.2 Fontes Secundárias (Recomendadas)

- [ ] **Artigos Técnicos**: Mínimo 2-3 artigos
  - [ ] Medium, Dev.to, HackerNoon
  - [ ] Blogs de especialistas
  - [ ] Artigos de empresas relevantes

- [ ] **Vídeos Educacionais**: Mínimo 1-2 vídeos
  - [ ] Canais educacionais confiáveis
  - [ ] Tutoriais oficiais
  - [ ] Conferências e webinars

- [ ] **Exemplos de Código**: Mínimo 1-2 exemplos
  - [ ] Repositórios GitHub verificados
  - [ ] Código funcional e atualizado
  - [ ] Licença compatível

### 2.3 Validação de Fontes

Para cada fonte, verificar:

- [ ] **Autoridade**: Fonte é confiável?
  - [ ] Autor é especialista reconhecido?
  - [ ] Organização é respeitada?
  - [ ] Site tem boa reputação?

- [ ] **Atualidade**: Conteúdo é recente?
  - [ ] Data de publicação verificada
  - [ ] Última atualização checada
  - [ ] Informações ainda são válidas?

- [ ] **Precisão**: Informações são corretas?
  - [ ] Fatos verificados em múltiplas fontes
  - [ ] Informações técnicas validadas
  - [ ] Exemplos testados (quando possível)

- [ ] **Relevância**: Fonte é relevante para o tópico?
  - [ ] Conteúdo relacionado diretamente
  - [ ] Exemplos práticos e aplicáveis
  - [ ] Nível de complexidade adequado

## Fase 3: Coleta de Recursos

### 3.1 Links e Documentação

- [ ] **Links Funcionais**
  - [ ] Todos os links testados
  - [ ] URLs são permanentes (evitar links temporários)
  - [ ] Links não requerem autenticação

- [ ] **Documentação**
  - [ ] Versão da documentação anotada
  - [ ] Seções específicas identificadas
  - [ ] Exemplos relevantes selecionados

### 3.2 Vídeos

- [ ] **Qualidade do Vídeo**
  - [ ] Áudio claro e compreensível
  - [ ] Conteúdo educativo e relevante
  - [ ] Duração apropriada (não muito longo)

- [ ] **Atualidade**
  - [ ] Vídeo é recente (últimos 2 anos preferencialmente)
  - [ ] Tecnologias mostradas ainda são atuais
  - [ ] Práticas demonstradas ainda são recomendadas

- [ ] **Acessibilidade**
  - [ ] Legendas disponíveis (preferencialmente)
  - [ ] Idioma compreensível
  - [ ] Qualidade de vídeo adequada

### 3.3 Código e Exemplos

- [ ] **Código Funcional**
  - [ ] Código testado (quando possível)
  - [ ] Exemplos completos e funcionais
  - [ ] Comentários explicativos presentes

- [ ] **Repositórios GitHub**
  - [ ] Repositório ativo (última atualização recente)
  - [ ] README completo e claro
  - [ ] Issues e PRs indicam manutenção ativa
  - [ ] Licença compatível

- [ ] **Documentação do Código**
  - [ ] Exemplos bem documentados
  - [ ] Instruções de uso claras
  - [ ] Dependências listadas

## Fase 4: Validação Cruzada

### 4.1 Verificação de Informações

- [ ] **Múltiplas Fontes**
  - [ ] Informações principais verificadas em 2+ fontes
  - [ ] Consenso entre fontes identificado
  - [ ] Discrepâncias investigadas e resolvidas

- [ ] **Informações Técnicas**
  - [ ] Definições verificadas em documentação oficial
  - [ ] Exemplos testados ou validados
  - [ ] Versões de APIs/ferramentas confirmadas

- [ ] **Informações Históricas/Conceituais**
  - [ ] Fatos históricos verificados
  - [ ] Datas e eventos confirmados
  - [ ] Contexto validado

### 4.2 Verificação de Atualidade

- [ ] **Tecnologias**
  - [ ] Versões atuais verificadas
  - [ ] APIs ainda suportadas
  - [ ] Práticas ainda recomendadas

- [ ] **Tendências**
  - [ ] Informações sobre tendências são recentes
  - [ ] Estatísticas atualizadas
  - [ ] Casos de uso contemporâneos

## Fase 5: Organização de Fontes

### 5.1 Documentação de Fontes

- [ ] **Lista de Fontes Criada**
  - [ ] Todas as fontes listadas
  - [ ] Tipo de fonte identificado
  - [ ] Data de acesso anotada

- [ ] **Recursos Selecionados**
  - [ ] 3-5 recursos mais relevantes escolhidos
  - [ ] Mix de tipos (link, video, document, code)
  - [ ] Recursos complementares entre si

### 5.2 Preparação para Inclusão

- [ ] **URLs Verificadas**
  - [ ] Todos os URLs funcionam
  - [ ] URLs são permanentes
  - [ ] URLs são acessíveis

- [ ] **Descrições Preparadas**
  - [ ] Descrições claras e informativas
  - [ ] Tipo de recurso identificado
  - [ ] Valor do recurso explicado

## Checklist de Qualidade das Fontes

### Critérios de Excelência

Uma fonte excelente deve ter:

- [ ] ✅ Autoridade reconhecida
- [ ] ✅ Informações atualizadas
- [ ] ✅ Conteúdo preciso e validado
- [ ] ✅ Relevância direta ao tópico
- [ ] ✅ Qualidade técnica alta
- [ ] ✅ Acessibilidade garantida
- [ ] ✅ Licença compatível (para código)

### Sinais de Alerta

Evitar fontes que:

- [ ] ❌ Não têm autor identificado
- [ ] ❌ São muito antigas (mais de 3 anos para tecnologias)
- [ ] ❌ Têm informações contraditórias com outras fontes
- [ ] ❌ São de sites não confiáveis
- [ ] ❌ Têm muitos erros técnicos
- [ ] ❌ Requerem pagamento ou autenticação
- [ ] ❌ Têm licenças restritivas

## Ferramentas Úteis

### Validação de Links
- [ ] Broken Link Checker usado
- [ ] Wayback Machine verificado (para links antigos)
- [ ] URLs testados manualmente

### Verificação de Código
- [ ] Código executado/testado
- [ ] Dependências verificadas
- [ ] Versões confirmadas

### Verificação de Vídeos
- [ ] Vídeo assistido completamente
- [ ] Conteúdo validado
- [ ] Qualidade verificada

## Notas Finais

- [ ] Todas as fontes documentadas
- [ ] Informações validadas cruzadamente
- [ ] Recursos selecionados e verificados
- [ ] Pronto para redação do conteúdo
